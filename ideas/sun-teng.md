# Ideas - Sun, Teng

## Initial Understanding

TOMI is an automotive company with two product lines: GEO for compact cars and TEO for sedan cars.

There is a trend among car manufacturers to apply IoMT technology and appealing software features to improve customer satisfaction.

George Wein is an employee of TOMI. He was inspired by several success stories, so he decide to develop a new model called iGEO for the GEO product line.

## Suggested WBS Items

### Prototype Design

George proposes the preliminary idea but it is not enough to be called a prototype.

To turn the initial idea into a prototype, more work from George and other employees need to do more work.

### Prototype Assessment

To ensure security and safety, we need to make sure that the software for IoMT features is decoupled from the GEO-specific software and hardware and that none of the IoMT features can influence driving-critical functions.

Therefore, we should involve as many relative employees as possible in assessing the prototype.

Also, since the iGEO Modules collect user data, we should assess their compliance.

### Market Survey

We should find out whether customers really like this idea, if so, which implement form they prefer

### Budget Management

After the prototype is approved, we should determine how many resources we should invest in the project.

This process will also require many meetings involving multiple employees.

We should not spend more than 1,000,000 euros.

### Development

Based on the prototype, the project can be divided into four parts:

- iGEO IoMT Hardware
- hosting application
- iGEO modules
  - WH Module
  - SC Module
- iGEO IoMT Display

We should identify the dependencies among these parts out so that we can arrange the development properly.

### Test

This includes unit test, integration test, system test and acceptance test and other necessary tests.

## Supplier Decision Ideas

I think choosing FunIOT is a good idea because:

- They charge lower prices, which is good for budget control
- They use an agile process and welcome requirement changes. Since this is a new and not yet mature idea, frequent requirement changes are inevitable.
- They have many young employees, many of whom have diverse backgrounds.

## Estimation Assumptions

Time: The project will be completed within one year.

Cost: The project will cost no more than 1,000,000 euros.

Every member of the project team has basic engineering, architecture, testing and management skills.

The selected supplier is capable of completing the assigned work.

## Scheduling/Resource Planning Ideas

We should wait for the Geoge's prototype, and then assess it.

After it has been assessed, we can carry out some works in parallel, such as market research, data compliance assessment, and supplier selection.

When all early-stage work is completed, we can start development. At this stage, we should assign assignments properly.

We should obtain information from the selected supplier frequently to ensure that the project stays on track.

We also need to inform stakeholders frequently, especially about new identified risks.

After development is completed, we start testing and then process the deployment.
