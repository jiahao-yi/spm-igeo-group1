# Contributing Guide

This repository is a shared workspace for Group 1. Keep contributions small, clear, and easy to merge into the final report.

## How to Contribute Ideas

1. Start with your own file in `ideas/`.
2. Write short but complete notes in English where possible.
3. Add reasoning, not only conclusions.
4. If you use information from the provided project documents, mention the relevant source document.

## Commit Messages

Use clear commit messages, for example:

```text
Add initial stakeholder observations
Update WBS version 1
Add supplier grid analysis draft
Refine planning poker estimates
```

## Before Merging Into the Report

- Check whether the idea fits the task requirements.
- Move mature content from `ideas/` into the matching file under `docs/`.
- Update the checklist in `README.md` when a section is ready for review.

## Files Not to Commit

Do not commit:

- `.DS_Store`
- Temporary Word or Office files
- Duplicate ZIP archives
- Personal notes unrelated to the submission
- Large generated files unless the group agrees they are needed

## Final Submission

The final submission should be one ZIP archive named `GROUP_1.zip`. It should contain the final PDF report and only the additional resources that are necessary for grading.
